# v0.1 Hardening Gate

This is the release-quality gate for `codex-memoryd` as a local-first memory
server with adapter surfaces. It is deliberately local and self-hosted: no CI
workflow exists in this repository today, so the checked-in script is the gate.

## One Command

```bash
scripts/v0.1-release-gate.sh
```

The default gate uses temp DBs and synthetic fixtures only. It does not read or
write `.dogfood/memory.db`.

Before tagging a v0.1 release from the live dogfood checkout, rerun with:

```bash
scripts/v0.1-release-gate.sh --include-dogfood
```

That opt-in lane exercises `.dogfood/`, Compose, native runtime smoke, read-only
MCP dogfood, and the write-capable sandbox lane.

## Required Local Checks

- `cargo fmt --all --check`
- `git diff --check`
- `cargo test`
- `codex-memoryd doctor --format json`
- `codex-memoryd eval substrate --compare --format json`
- `codex-memoryd eval procedures --format json`
- `codex-memoryd perf --format json`
- `scripts/demo-substrate.sh --dry-run`
- `scripts/dogfood-write-sandbox.sh --dry-run`

The script writes reports under `target/release-gate/` by default, including
doctor, eval, perf, step logs, and a machine-readable summary.

## Tag-Required Operator Checks

Run these before tagging from a checkout that has local dogfood available:

- `scripts/dogfood-compose-heartbeat.sh`
- `docker compose down`
- `scripts/codex-memoryd-local-runtime.sh start`
- `scripts/codex-memoryd-local-runtime.sh status`
- `scripts/codex-memoryd-local-runtime.sh smoke`
- `scripts/codex-memoryd-local-runtime.sh restart-survival`
- `scripts/codex-memoryd-local-runtime.sh stop`
- `scripts/dogfood-write-sandbox.sh run`

These checks may touch `.dogfood/`, so they are not part of the default script
path used by agents in isolated worktrees.

## Non-Goals

- No hosted service.
- No dashboard.
- No new memory primitive.
- No automatic promotion from sandbox to real dogfood DB.
- No scheduler or Dreamer auto-apply requirement.

## Caveats

- GitHub Actions is not configured; local script output is the source of truth.
- The default gate skips live dogfood because isolated worktrees often do not
  have `.dogfood/memory.db`.
- Release artifacts must remain content-free. Do not commit generated DBs,
  dogfood exports, sandbox diff reports containing local ids, or real-memory
  snapshots.
